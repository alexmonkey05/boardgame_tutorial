# 보드게임 DB 관리자 페이지 프롬프트

당신은 FastAPI 관리자 API와 운영자용 웹 콘솔을 함께 설계하는 개발자입니다.

## 목적

관리자 페이지는 보드게임 마스터 데이터의 정확성과 이미지 식별 품질을 관리하는 내부 도구입니다. 업무 화면답게 밀도 있고 예측 가능하게 구성하며, 데이터 변경 전 확인 단계를 둡니다.

## 접속과 인증

- 경로: `/admin-ui`
- 인증: Railway Variables의 `ADMIN_TOKEN`
- 요청 헤더: `X-Admin-Token`
- 브라우저 저장: `sessionStorage`만 사용
- URL, 로그, DOM 고정값에 토큰을 포함하지 않음
- 인증 실패 시 토큰 재입력 화면으로 복귀

## 메뉴

1. 상태: `/health`, `/meta/schema`, DB 경로와 Vision 설정 여부
2. 게임 데이터: 목록, 검색, 생성, 수정
3. 데이터 품질: 별칭과 게임 관계 추가·삭제
4. 운영 로그: 최근 인식, 추천, 사용자 이벤트 요약

## 게임 편집

다음 필드를 관리합니다.

- ID, 한국어·영문 이름
- 짧은 설명과 규칙 요약
- 최소·최대 인원, 평균 시간, 난이도
- 장르, 태그, 플레이 스타일
- 초보자·아이·파티·전략 특성
- 이미지 URL

새 게임 생성과 기존 게임 수정은 구분하고, 수정 시 변경 대상을 확인 모달에 보여줍니다.

## 별칭과 관계

- 별칭은 정규화 후 중복을 거부합니다.
- 관계는 대상 게임과 유형을 선택해 생성합니다.
- 자기 자신을 대상으로 한 관계와 중복 관계를 거부합니다.
- 삭제는 확인 모달을 거친 후 수행합니다.

## 로그 안전성

로그 화면에는 시간, 상태, 게임 ID, 후보 수, 확정 여부와 요청 조건 요약만 표시합니다. 다음 값은 반환하거나 렌더링하지 않습니다.

- 관리자 토큰
- Vision API 키와 endpoint 원문
- 이미지 base64
- 외부 Vision API 원문 응답
- 사용자 자유 입력 payload 값

## 완료 기준

- 관리자 토큰 없이는 관리자 API를 호출할 수 없습니다.
- 게임 생성·수정, 별칭·관계 생성·삭제가 동작합니다.
- 최근 인식·추천·이벤트 로그를 안전하게 조회합니다.
- 위치별 보유 데이터 관리 기능이나 관련 API가 없습니다.
- 관리자 API 자동 테스트가 통과합니다.
