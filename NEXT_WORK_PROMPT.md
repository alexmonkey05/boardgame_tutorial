# 다음 작업 프롬프트: 보드게임 마스터 DB 품질과 운영 안정성 강화

당신은 FastAPI, 데이터 품질 관리, PostgreSQL 마이그레이션과 운영 자동화를 함께 설계하는 시니어 개발자입니다.

현재 서비스는 위치별 보유 데이터 의존성을 제거했고, 보드게임 식별·검색·추천 서비스로 전환됐습니다. 사용자 화면, 추천, 인식, 관리자 API와 문서는 게임 마스터 DB 중심으로 동작하며 자동 테스트가 통과합니다.

## 다음 목표

다음 단계는 샘플 7개 게임 수준의 MVP를 운영 가능한 보드게임 데이터 서비스로 확장하는 것입니다.

1. 게임 마스터 데이터의 품질 규칙을 정의합니다.
2. 관리자 CSV 가져오기·내보내기를 구현합니다.
3. 중복 게임, 별칭 충돌, 끊어진 관계를 탐지합니다.
4. 실제 게임 이미지 출처와 라이선스 메타데이터를 관리합니다.
5. SQLite Volume 백업·복원 절차를 검증합니다.
6. PostgreSQL 전환 계획과 리허설 도구를 준비합니다.
7. 공개 API의 rate limit과 기본 관측성을 추가합니다.

## 데이터 품질 규칙

- 게임 ID는 안정적인 slug이며 변경하지 않습니다.
- 한국어 이름, 설명, 규칙 요약, 인원, 시간, 난이도, 장르는 필수입니다.
- 최소 인원은 최대 인원보다 클 수 없습니다.
- 시간과 인원은 양수여야 합니다.
- 난이도와 장르는 허용된 값 집합을 사용합니다.
- 동일 게임의 정규화된 별칭은 중복될 수 없습니다.
- 하나의 별칭이 여러 게임에 연결되면 충돌로 표시합니다.
- 자기 관계와 중복 관계를 금지합니다.
- 이미지에는 URL, 출처, 라이선스, 대체 텍스트를 함께 관리합니다.

## 관리자 기능

추가할 기능:

- CSV 미리보기, 검증, 오류 행 표시
- 유효한 행만 적용하거나 전체 취소하는 명시적 선택
- 기존 데이터와의 생성·수정 diff
- 게임과 별칭 CSV 내보내기
- 데이터 품질 대시보드
- 별칭 충돌과 끊어진 관계 해결 흐름
- 변경 작업 감사 로그

모든 변경 작업은 `X-Admin-Token` 인증, 확인 단계와 민감값 비노출 원칙을 유지합니다.

## API 후보

```text
GET /admin/data-quality
POST /admin/imports/games/preview
POST /admin/imports/games/apply
GET /admin/exports/games.csv
GET /admin/audit-events
```

CSV 적용 API는 preview 결과에서 발급한 짧은 수명의 import ID를 요구하고, 원본 파일이나 토큰을 로그에 남기지 않습니다.

## 저장소 전환 준비

- 현재 SQLite 스키마와 PostgreSQL 목표 스키마를 문서화합니다.
- SQLite 데이터를 읽어 PostgreSQL에 적재하는 idempotent 스크립트를 만듭니다.
- 행 수, 주요 ID, 외래키와 JSON 필드를 검증합니다.
- dry-run과 실제 적용 모드를 분리합니다.
- 실패 시 기존 Railway 서비스를 계속 사용할 수 있어야 합니다.
- 실제 전환은 백업과 복원 리허설이 성공한 뒤 별도 승인으로 진행합니다.

## 운영 안정성

- 이미지 업로드와 추천 API에 사용자 친화적인 rate limit을 검토합니다.
- 요청 ID, 처리 시간, 상태 코드와 외부 Vision 성공 여부를 구조화 로그로 남깁니다.
- 토큰, API 키, 이미지 바이트, 자유 입력 원문은 로그에서 제외합니다.
- Healthcheck와 readiness를 구분할 필요가 있는지 검토합니다.
- 외부 Vision 장애율과 fallback 비율을 확인할 수 있게 합니다.

## 테스트

- CSV 정상·오류·중복·충돌 행
- preview 없이 apply 불가
- 관계 무결성과 별칭 전역 충돌
- 감사 로그의 민감값 비노출
- SQLite→PostgreSQL dry-run의 행 수 검증
- rate limit 경계와 기존 개인정보 삭제 회귀
- 관리자 UI 모바일·데스크톱 레이아웃

## 완료 기준

- 운영자가 CSV를 안전하게 검증하고 적용할 수 있습니다.
- 데이터 품질 문제를 관리자 화면에서 발견하고 해결할 수 있습니다.
- 이미지 출처와 라이선스를 추적할 수 있습니다.
- SQLite 백업·복원 리허설 결과가 문서화됩니다.
- PostgreSQL 전환 스크립트가 dry-run에서 검증됩니다.
- 공개 API에 필요한 최소 운영 보호 장치가 적용됩니다.
- 자동 테스트와 Railway 배포 검증이 통과합니다.

## 주의사항

- 운영 Volume 데이터를 자동 삭제하거나 파괴적으로 마이그레이션하지 마세요.
- PostgreSQL 실제 전환, Volume 정리와 대규모 데이터 적용은 사용자 승인 없이 수행하지 마세요.
- 이미지 URL을 추가할 때 출처와 이용 조건을 확인하세요.
- 데이터 양을 늘리기 전에 중복과 라이선스 정책을 먼저 고정하세요.
