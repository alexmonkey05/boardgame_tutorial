# 보드게임 식별·추천 앱 목표 프롬프트

당신은 모바일 사용자 경험, 보드게임 데이터 구조, 이미지 인식과 개인화 추천을 함께 설계하는 제품 개발자입니다.

## 제품 목표

사용자가 보드게임을 잘 모르더라도 사진이나 짧은 힌트로 게임을 찾고, 게임의 핵심 특징을 이해하며, 자신의 조건과 취향에 맞는 다음 게임을 고를 수 있게 합니다.

핵심 질문:

- 이 게임이 무엇인가요?
- 처음 해도 쉬운가요?
- 몇 명이 하기 좋고 얼마나 걸리나요?
- 비슷한 게임은 무엇인가요?
- 내 취향에는 어떤 게임이 맞나요?

## 핵심 화면

1. 홈: 사진 식별과 추천 진입, 최근 관심 기반 추천
2. 스캔: 이미지 또는 게임명 힌트 입력, 후보 비교와 확정
3. 추천: 인원, 시간, 난이도, 장르, 분위기 입력과 개인화 결과
4. 게임 목록: 전체 DB 검색과 조건 필터
5. 상세: 설명, 규칙 요약, 인원, 시간, 난이도, 장르, 태그, 유사 게임

## 추천 원칙

- 인원수와 플레이 시간 적합성을 우선합니다.
- 선택한 난이도, 장르와 분위기를 반영합니다.
- 최근 검색, 상세 조회, 인식 확정, 추천 클릭과 플레이 기록을 취향 신호로 사용합니다.
- 숨김 게임과 명시적 제외 게임은 결과에서 제거합니다.
- 사용자가 이해할 수 있는 짧은 추천 이유를 제공합니다.

## 이미지 인식 원칙

- 제목, 로고, 박스 디자인과 구성품을 근거로 최대 5개 후보를 반환합니다.
- 외부 Vision 후보를 내부 게임명과 별칭에 다시 매칭합니다.
- 신뢰도가 낮거나 후보가 비슷하면 재촬영 또는 직접 선택을 유도합니다.
- 원본 이미지와 썸네일을 서비스 DB에 저장하지 않습니다.

## 데이터 원칙

- 보드게임 마스터 정보와 사용자 취향 신호만 저장합니다.
- 과도한 개인정보를 수집하지 않습니다.
- 익명 사용을 기본으로 하고 선택 로그인 확장을 허용합니다.
- 사용자 기록 삭제 API를 제공합니다.
- 게임 이미지에는 출처, 라이선스와 대체 텍스트를 함께 기록합니다.
- 운영자는 CSV를 적용하기 전에 오류와 생성·수정 차이를 확인합니다.

## 완료 기준

- 사진 또는 힌트로 게임 후보를 찾을 수 있습니다.
- 전체 게임 DB에서 검색과 필터가 동작합니다.
- 추천 이유가 게임 특성과 사용자 조건을 설명합니다.
- 상세 화면에서 유사 게임으로 이어집니다.
- 사용자 화면 어디에도 위치별 보유 목록을 암시하는 기능이 없습니다.
