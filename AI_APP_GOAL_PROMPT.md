# 보드게임 카페 모바일 앱 목표 설명 프롬프트

당신은 모바일 앱 기획자이자 제품 전략가입니다. 아래 목표를 바탕으로, 보드게임 카페 이용자가 현장에서 바로 사용할 수 있는 모바일 앱의 핵심 기능, 사용자 흐름, 화면 구조, 데이터 요구사항, AI 활용 방식, MVP 범위를 구체화해주세요.

## 앱의 핵심 목표

이 앱은 보드게임 카페에서 사용자가 "어떤 게임을 하지?", "이 게임이 뭔지 알아?", "여기에는 어떤 게임들이 있지?"라고 고민하는 순간을 빠르게 해결하기 위한 모바일 앱입니다.

사용자는 보드게임에 익숙하지 않을 수도 있고, 같이 온 사람들의 취향이나 플레이 시간, 난이도, 인원수에 맞는 게임을 찾기 어려워할 수 있습니다. 앱은 이런 상황에서 사용자의 선택 부담을 줄이고, 현재 있는 보드게임 카페 안에서 바로 플레이할 수 있는 게임을 찾도록 도와야 합니다.

## 주요 기능 목표

### 1. 카메라 기반 보드게임 식별

사용자가 보드게임 박스나 구성품을 카메라로 촬영하면, 앱은 사진 속 보드게임이 무엇인지 판단합니다.

앱 자체가 보드게임의 상세 설명을 모두 생성할 필요는 없습니다. 보드게임 설명, 규칙 요약, 난이도, 플레이 시간, 추천 인원, 장르 등의 정보는 서버에 저장되어 있다고 가정합니다. 따라서 앱의 AI 역할은 사진 속 보드게임을 식별하고, 서버에 저장된 해당 보드게임 데이터를 불러오는 데 필요한 식별 결과를 제공하는 것입니다.

카메라 인식 결과는 항상 확률과 대체 후보를 함께 고려해야 합니다. 사용자가 촬영한 사진이 흐리거나 박스 일부만 보이거나, 비슷한 디자인의 확장판이 있을 수 있으므로 "가장 가능성 높은 게임", "후보 게임 목록", "다시 촬영 필요" 같은 상태를 설계해주세요.

### 2. 이전 플레이 및 검색 기록 기반 추천

앱은 사용자가 이전에 플레이했거나 검색했던 보드게임 기록을 바탕으로 새로운 보드게임을 추천합니다.

추천은 단순히 인기순이 아니라 사용자의 취향을 반영해야 합니다. 예를 들어 사용자가 자주 찾은 게임의 장르, 난이도, 플레이 시간, 참여 인원, 협력/경쟁 성향, 전략성, 파티성 등을 바탕으로 추천 이유를 함께 제시해야 합니다.

추천 결과는 현재 사용자가 있는 보드게임 카페의 보유 게임 목록과 연결되어야 합니다. 사용자가 마음에 드는 추천을 받더라도 해당 카페에 없는 게임이면 현장에서 바로 플레이하기 어렵기 때문입니다. 따라서 "내 취향에 맞고, 지금 이 카페에 있는 게임"을 우선 추천하는 방향으로 설계해주세요.

### 3. 현재 보드게임 카페의 보유 게임 목록 제공

사용자가 현재 방문 중인 보드게임 카페의 보드게임 리스트를 확인할 수 있어야 합니다.

보유 게임 목록은 검색, 필터, 정렬이 가능해야 합니다. 사용자는 인원수, 예상 플레이 시간, 난이도, 장르, 초보자 추천 여부, 아이와 함께 가능 여부, 파티 게임 여부, 전략 게임 여부 등을 기준으로 게임을 좁혀볼 수 있어야 합니다.

가능하다면 사용자의 현재 위치, QR 코드, 매장 선택, 또는 카페별 고유 식별자를 통해 현재 매장을 인식하는 흐름을 제안해주세요.

## 사용 상황

앱은 보드게임 카페 안에서 빠르게 쓰이는 것을 목표로 합니다. 사용자는 친구, 연인, 가족, 동료와 함께 있을 수 있고, 오래 읽기보다 빠르게 선택하고 싶어합니다.

따라서 화면과 문구는 짧고 직관적이어야 합니다. 사용자는 게임 전문가가 아닐 수 있으므로, 전문 용어보다 "처음 해도 쉬움", "대화가 많이 필요한 게임", "머리를 많이 쓰는 게임", "30분 안에 끝남", "4명이 하면 재밌음"처럼 실제 선택에 도움이 되는 표현을 우선 사용해야 합니다.

## 중요한 설계 원칙

- 사용자는 보드게임을 잘 모를 수 있으므로 쉬운 언어를 사용합니다.
- 보드게임 설명 데이터는 서버에 저장되어 있고, 앱은 식별과 추천 경험에 집중합니다.
- 카메라 인식은 단일 정답만 강요하지 말고 후보와 신뢰도를 함께 보여줍니다.
- 추천은 사용자의 이전 플레이 및 검색 기록과 현재 카페의 보유 게임 목록을 함께 고려합니다.
- 앱은 보드게임 카페 현장에서 빠르게 쓰이는 도구여야 하므로, 선택까지 걸리는 시간을 최소화합니다.
- 사용자가 함께 온 사람들의 인원수, 남은 시간, 원하는 분위기를 쉽게 입력할 수 있어야 합니다.
- 초보자도 부담 없이 사용할 수 있는 친절한 UX를 우선합니다.
